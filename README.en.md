# MegaForm

[简体中文](README.md) | [English](README.en.md)

**A multi-model tree conversation system for research-style questioning.**

MegaForm turns "one question -> parallel answers from multiple models -> branch deeper from a chosen answer" into an expandable, collapsible, deep-linkable conversation tree.

## Why MegaForm

Most chat tools are linear: you ask, the model answers, and follow-ups keep piling into the same context. That breaks down when you need comparison, branching, and backtracking.

| Scenario | Linear chat | MegaForm |
|---|---|---|
| Ask multiple models the same question | Open multiple windows and compare manually | One question, multiple model answers side by side |
| Follow up on one model's answer | Context gets mixed with other answers | The follow-up inherits only the current branch context |
| Dig into a selected sentence | Copy/paste and hope the model remembers | Nut anchors preserve the selected text and inject context |
| Review old branches | Scroll around and guess which path is which | A visible tree with folding and focused navigation |

A typical question tree looks like this:

```text
Root question: "What is the core difference between RLHF and DPO?"
├─ GPT-5 answer
│  └─ Follow-up: "How does the reward model avoid reward hacking in RLHF?"
│     └─ Follow-up: "What alternatives to PPO exist in RLHF?"
├─ Claude Sonnet 4.6 answer
│  └─ Follow-up: "How is DPO's implicit reward formula derived?"
└─ DeepSeek V4 Pro answer
```

## Core Features

### Parallel Multi-Model Chat

- Send one question to multiple models at once and compare answers side by side.
- Includes presets for OpenAI, Anthropic, Gemini, xAI (Grok), OpenRouter, DeepSeek, Zhipu AI, MiniMax, Kimi, Qwen, and Ollama.
- Supports local Ollama models and arbitrary OpenAI-compatible endpoints.
- Per-conversation model selection, thinking depth, web search, and Profile injection.

### Tree-Shaped Conversations

- Data model: `Root Node -> Node -> Response -> Nut`.
- Progression branches continue a line of thought.
- Follow-up branches target selected text from an answer.
- The tree supports folding, focused mode, breadcrumbs, and deep links.

### SSE Streaming

- Backend LLM calls are decoupled from HTTP connections with `asyncio.create_task`.
- If the frontend disconnects, background generation continues and reconnects can restore from SQLite.
- Responses are throttled into the database to keep reconnects smooth.
- Large trees can be streamed progressively with BFS loading.

### Text Anchors (Nuts)

- Select any passage in a response, ask a follow-up, and MegaForm injects the selected passage into the next prompt.
- Follow-up cards appear near the referenced text.
- Matching uses direct search, Markdown-normalized search, and word-level fallback.

### Web Search

- Search backends: Brave, Serper, Tavily, SerpAPI, and SearXNG.
- Native model search: Anthropic web search, Gemini grounding, and OpenAI search-preview models.
- Tool-calling search: models can call `search_web` and `see_web` for up to 7 tool rounds.
- Search sources are shown in the frontend.

### Reasoning / Thinking

- Per-model thinking budgets and visual strength controls.
- Streaming thinking output with collapsible display.
- Provider-specific parameter adaptation for Anthropic extended thinking, DeepSeek reasoning, Gemini thinking config, and OpenAI reasoning.

### Usage And Cost Tracking

- Records input/output tokens, latency, model, and provider for each response.
- Estimates cost from per-model pricing.
- Falls back to character-based token estimates when providers do not return usage.
- Supports CNY and USD pricing units.

### Auto Summaries

- Node summaries can be edited manually or generated by a configured summary model.
- Root summaries refresh after quiet periods and during the daily scan.
- Summaries make large question trees easier to browse.

### Auth And Profiles

- Local mode by default for single-user use.
- Optional email/password login.
- Optional Google OAuth.
- Shared model configs allow family/team members to share encrypted API keys while tracking usage separately.
- Markdown Profile content can be injected into the system prompt as background preference/context.

## Quick Start

### Requirements

- Python 3.10+
- Node.js 18+
- npm

### 1. Install

```bash
cd ~/projects/megaform

pip install -r requirements.txt

cd frontend && npm install && cd ..
```

### 2. Development

Open two terminals:

```bash
python main.py
```

```bash
cd frontend && npm run dev
```

Visit `http://localhost:5173`. Vite proxies `/api` to the backend on port `8080`.

### 3. Production

```bash
cd frontend && npm run build && cd ..
python main.py
```

Visit `http://localhost:8080`.

The SQLite database `megaform.db` is created and migrated automatically in the project root. If no model config exists, MegaForm creates an empty DeepSeek Chat placeholder.

## Configuration

### Environment

Copy the example file and edit it for your machine:

```bash
cp .env.example .env
```

Key variables:

```bash
MEGAFORM_AUTH_MODE=local
MEGAFORM_EMAIL_AUTH=true
MEGAFORM_SECRET_KEY=
MEGAFORM_PUBLIC_BASE_URL=http://localhost:8080
MEGAFORM_COOKIE_SECURE=false

GOOGLE_CLIENT_ID=
GOOGLE_CLIENT_SECRET=
GOOGLE_REDIRECT_URI=http://localhost:8080/api/auth/google/callback
```

Use a stable `MEGAFORM_SECRET_KEY` in production. Changing it can make encrypted API keys unreadable.

### Model Configs

Open the settings modal, choose a provider preset, enter an API key, and add models. Important fields:

| Field | Meaning |
|---|---|
| `name` | Display name |
| `provider` | Provider type such as `openai`, `deepseek`, `ollama`, `anthropic`, or `custom` |
| `base_url` | API base URL |
| `api_key` | API key, encrypted at rest |
| `model_name` | Provider-side model id |
| `max_tokens` | Maximum output tokens |
| `price_per_input` | Input price per 1K tokens |
| `price_per_output` | Output price per 1K tokens |

### Web Search

Configure a search provider in Settings -> Web search.

| Provider | Free tier | Paid pricing |
|---|---|---|
| Brave Search | 2,000/month | $5/1K searches |
| Serper | 2,500/month | $0.30/1K searches |
| Tavily | 1,000/month | $0.008/search |
| SerpAPI | 100/month | From $50/month |
| SearXNG | Free | Self-hosted |

### Language

The interface supports Simplified Chinese and English. The selected language is saved to the current user and is also used for backend-generated model prompts, summaries, and web-search tool text.

## Data Model

```text
nodes (root nodes are question trees)
  ├─ responses
  │   └─ nuts
  └─ child nodes
```

Important constraints:

- Deleting a root node cascades through the whole question tree.
- Deleting a non-root node cascades through that node and its descendants.
- Model configs use soft deletion so historical answers and usage remain readable.
- FTS5 indexes cover node content, node summaries, and response content.

## API Overview

| Method | Path | Description |
|---|---|---|
| `GET` | `/api/roots` | List question trees |
| `GET` | `/api/roots/{id}` | Read a root node |
| `PATCH` | `/api/roots/{id}` | Update root content, summary, pin/archive state |
| `DELETE` | `/api/roots/{id}` | Delete a question tree |
| `GET` | `/api/roots/{id}/tree` | Read the whole tree |
| `GET` | `/api/roots/{id}/tree/stream` | Stream tree loading with SSE |
| `GET` | `/api/nodes/{id}` | Read a node with responses |
| `PATCH` | `/api/nodes/{id}` | Update a node |
| `DELETE` | `/api/nodes/{id}` | Delete a node and descendants |
| `POST` | `/api/nodes/{id}/generate-summary` | Generate a node summary |
| `POST` | `/api/nodes/{id}/rerun/stream` | Rerun a node with SSE |
| `POST` | `/api/chat/stream` | Create a node and stream multi-model answers |
| `GET` | `/api/chat/stream/{node_id}` | Reconnect to node streaming |
| `POST` | `/api/node/{node_id}/add-model` | Add a model answer to an existing node |
| `GET` | `/api/search?q=...` | Full-text search |
| `GET` | `/api/models` | List model configs |
| `POST` | `/api/models` | Create or update model configs |
| `GET` | `/api/settings` | Read settings |
| `POST` | `/api/settings` | Save settings |
| `GET` | `/api/token-usage` | Usage and cost stats |

SSE endpoints emit events such as `node_ready`, `model_start`, `thinking`, `content`, `sources`, `model_done`, `model_error`, and `done`.

### Notes

- Vite can prefer stale generated `.js` files over `.tsx`; clean generated frontend source artifacts before release builds if needed.
- SSE `onDone` handlers should be guarded with `try/finally` so UI state does not get stuck.
- Gemini direct calls use the native Google API; OpenRouter and other proxies remain OpenAI-compatible.
